「Webコンテンツの作成」の成果

本テーマについて

先行する情報工学の基礎知識を学び専門的なことだけでなくレポートからプレゼンテーションといった流れや コミュニケーションをすることの大切さを学ぶと同時に課題に対する解決能力や重要性の理解,Web作成などができるようになりたい。

プロジェクト課題について

プロジェクト課題で作成した課題で作成したページ:project.html

プロジェクト課題では様々な国についての天気予報を調べれるページを作りました。このページでは検索欄に表示されている国名を指定すると、その国の首都や気温や天気、緯度経度などを表示してくれます。 project.htmlはkの授業の中で一番難しかったと思います。javascriptの内容を変えずにどうやってhtmlに複数の国の情報を反映させるのかが分からなかったことや検索ツールを作成することがとても大変でした。

数当てゲーム

数当てゲームのページ:kazuate.html

数当てでは1〜10の乱数を生成し、3回目までに正解、4回目以降に正解すると答えとゲーム終了を告げるメッセージが表示されるようなプログラムを作りました 仕組みは単純で入力した値と乱数で生成された値を条件式で判断し、それに対応した文字列を表示するという内容だったけれど、ボタンを配置、htmlに書かれている内容を上書きすることや、新しく習ったことばかりを使っていたので、 作成をするのはかなり大変だった。それだけでなく、入力をするごとに前回の結果で表示した文字列を消し、新たな文字列を表示させなければならなかったので作成するのにはかなり混乱しました。 自分でWebサイトを作り、簡単なゲームを作ることができたことはとてもいい体験だと思いました。

各授業回の学習事項

第1回 WWW と HTML

第1回のWWWとHTMLではWebサイトを作るにあたって必要最低限なタグの使い方を学び実際に使用することでより深く理解をしていきました。 ハイパーリンクや画像の貼り付けなどといった少しでも達成感を得られる内容を学びました。これらのことを活かして、この授業での最終も目標であるWeb作成 を完成させたいです。

演習の成果:演習1-4: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

第2回の講義ではCSSというWebページの装飾を学びました。cssプログラムでhtmlで使ったタグを指定し、コマンドを打ち込むことでページの装飾ができるといったものでした。CSSを学んだことでWebページの雰囲気が一気に華やかになったと思います。

演習の成果:演習2-3: ページの見た目の装飾

第3回 JavaScript の基本

第3回の授業ではJavaScriptを学びました。今までやっていたjavaと少し類似している点があったけれど実際にやってみると似ているのは形だけで内容はかなり違ったように感じました。 そして第3回からはJavaScriptを使い本格的に数当てゲームの作成が始まりました。

演習の成果:演習3-6: JavaScriptの使い方

第4回 DOM と JavaScript

第4回の講義では、JavaScriptの続きと新しくDOMを学びました。DOMとはJavaScriptと合わせることで動きのあるWebページを作ることができ、要素の追加、消去や要素の変更などができるようになった。

演習の成果:演習4-2~5: DOMとJavaScript

第5回 イベント処理

第5回の講義ではイベント処理ということを学びました。イベント処理とはユーザーが操作すると特定の動作が起き、Webページに躍動感を与えてくれる要素だと思います。数当てゲームやproject課題にも用いた必要不可欠な内容でした。

演習の成果:演習5-3: イベント処理

授業の感想

第1回から第5回までの内容を使いindex,htmlではWebページの装飾をなるべく見やすい色や配置に編集しました。また、indexページから各サイトに飛んだ後indexページに 戻るためのハイパーリンクを設置しました。 各サイトではtargetが_blankで指定されているので新しくタブを開いてしまいますが簡単に元のページに戻れるようにしました そしてこの授業の苦心点はDOMです。DOMではjavascriptと一緒に使うことでWebページをより効率的に作成するために必要不可欠な要素でしたが、プログラミングに少し苦手意識があるので 同じくDOMやjavascriptに少し苦戦しました。 また、数当てゲームのサイトを作る際に、入力した値をhtmlに反映させることや入力した値によって表示する文字列を変えるということが難しかったです。 プログラミングでは何回もやっていることなのでやりたいことはわかっていたけれど、javascriptになると一気に難易度が上がった気がしました。強調点として見てほしいのはindex.htmlとproject.htmlです。 index.htmlでは誰が見ても見やすい色で装飾しており、配置も見やすいようにしてあり初めてみる人にとって見やすいという良い印象を与えると思います。そして、project.htmlでは検索キーのところを ユーザーが入力をしなくても最初から選択肢が与えられ、知りたい地域の天気や気温をいち早くわかるようにし、ユーザーの利便性を高めました。6回という短い期間での授業でしたが、初めて自分のWebサイトを作れた時はかなり嬉しかったし、 達成感を感じることができました。javascriptやDOMは難しいと感じることが多く試行錯誤しながらWebページを作り上げていくのには非常に充実していました。、逆にSCCやHTMLを編集し、装飾をすることは学んでいてもかなり楽しいと思えたし、 自分から進んでタグやプロパティを調べたりし、所々にそれを応用として使っているので楽しみながらサイトを作り上げることができました。それだけではなく、CSSとはデザイン性が大事になってきて間違いという概念がないため、 どんどん新しいことに挑戦できるということにやりがいを感じました。サイトを作るという点ではとても楽しかったと思います。今後も学びを深めていきたいと思います。
